# My 90-Day DevOps & Cloud Learning Plan (2026)

**Name:** Said Numan Bacha 
**Current Level:** Full Stack Engineer transitioning into DevOps & Cloud Engineering  
**Start Date:** June 23, 2026

### My Understanding of DevOps & Cloud Engineering
DevOps is the bridge between Development and Operations — a culture + set of practices focused on automation, collaboration, and faster, more reliable software delivery.  
Cloud Engineering involves designing, building, and managing scalable infrastructure on platforms like AWS. Together, they enable end-to-end ownership: from writing code to deploying and running it successfully in production.

### Why I Am Starting This Journey
I want to move to the next level in my career. As a Full Stack Engineer, I have good experience building applications, but I lack visibility into the full lifecycle — especially what happens **after** the code is written (deployment, scaling, monitoring, troubleshooting) and the **system/cloud design** decisions made before development.  
This transition will make me a more complete, high-value engineer.

### My 90-Day Goals

**Big Career Goal:** Successfully transition from Full Stack Engineer to DevOps + Cloud Engineer / System Engineer.

**Specific Goals:**
1. Deploy **5–6 production-grade applications** on Kubernetes (with CI/CD, monitoring, and proper networking).
2. Master Linux troubleshooting, automation (Shell scripting + CI/CD pipelines), and build/debug real Kubernetes clusters.
3. Build strong foundational + practical knowledge of the entire DevOps toolchain (Linux → Git → Docker → K8s → Terraform → Jenkins/Ansible) and understand cloud architecture.

**Core Skills to Build:**
- Linux mastery & troubleshooting
- CI/CD pipeline automation
- Kubernetes (deployment, debugging, orchestration)

### Weekly Time Commitment (Realistic & Sustainable)
